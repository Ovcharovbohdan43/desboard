# Управление командами, клиентами и правами доступа

## Обзор

- **Команды (teams)** — рабочее пространство (агентство, студия).
- **Участники команды (team_members)** — пользователи с ролями.
- **Клиенты (clients)** — принадлежат команде; проект связывается с клиентом.
- **Проекты (projects)** — принадлежат команде, опционально привязаны к клиенту.

---

## Роли в команде

| Роль     | Описание                                 | Права                                                     |
|----------|-------------------------------------------|-----------------------------------------------------------|
| **owner**  | Владелец команды                          | Полный контроль, управление участниками, настройками      |
| **admin**  | Администратор                             | Добавление/удаление участников, смена ролей, настройки    |
| **member** | Участник (по умолчанию)                   | Работа над проектами, управление клиентами                |
| **guest**  | Гость                                     | Просмотр (в будущем — read-only)                          |

- Создатель команды автоматически получает роль `owner` в `team_members`.
- Роли проверяются через RLS; UI показывает «Manage team» только владельцам и администраторам.

---

## Связь клиентов с командами

- Каждый клиент привязан к одной команде (`clients.team_id`).
- Проект может быть связан с клиентом (`projects.client_id`).
- Участники команды видят только клиентов своей команды.
- Клиент не имеет отдельного логина; доступ к Client Portal — по ссылке (UUID/slug проекта).

---

## Client Portal и права

- **Внутренний портал** (`/portal/:portalId`): требует авторизации, доступ есть у участников команды.
- **Внешний портал** (`/client/:projectId`): два режима:
  - **С токеном** (`/client/:id?token=xxx`): анонимный доступ — клиент может просматривать, апрувить, запрашивать правки, ставить рейтинг, отправлять сообщения, отмечать deliverables без входа. Кнопка «Copy Client Link» в виджете генерирует ссылку с токеном (срок 30 дней).
  - **Без токена** (авторизованный): клиент входит в аккаунт команды — полный доступ, включая загрузку файлов.

---

## Управление участниками

- В селекторе команды доступен пункт **«Manage team»** для owner/admin.
- В диалоге можно:
  - **Переименовать команду** — кнопка «Rename» (owner/admin).
  - **Пригласить по email** — ввод email, выбор роли (admin/member/guest), копирование ссылки `/invite/:token`.
  - Просматривать ожидающие приглашения, копировать ссылку, отменять.
  - Просматривать участников и роли; менять роль (owner/admin).
  - Удалять участников или выходить из команды.
- Нельзя удалить или понизить последнего владельца (триггер в БД).
- **Принятие приглашения**: пользователь переходит по `/invite/:token`. Должен быть залогинен с тем же email. После успеха — редирект на dashboard с выбранной командой.
