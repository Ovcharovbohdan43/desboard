# Обзор продукта — Desboard Hub

**Версия:** 1.0  
**Дата:** 2026-03-11

---

## 1. Для кого эта платформа

**Desboard Hub** — это **хаб для команд** (студии, агентства, отделы), которые ведут проекты для клиентов: дизайн, разработка, консалтинг и т.п.

**Ключевые сущности:**
- **Команда (Team)** — рабочая единица; у пользователя может быть несколько команд.
- **Проекты** — привязаны к команде и опционально к клиенту; внутри: задачи, майлстоуны, деливерабли, файлы.
- **Клиенты** — контрагенты команды; доступ к порталу по ссылке с токеном (без логина).
- **Деливерабли** — что передаётся клиенту (версии, статусы, обратная связь).
- **Портал клиента** — внешняя страница для клиента: просмотр проектов/деливераблов, сообщения, апрувы.

То есть платформа закрывает цикл: **внутренняя работа команды** (проекты, задачи, файлы, финансы) + **передача результата клиенту** (деливерабли, портал, обратная связь).

---

## 2. Три уровня интерфейса

### 2.1 Главная страница (Dashboard, `/`)

**Смысл:** Одна страница с **карточками-виджетами**. Каждая карточка — краткий превью (счётчики, последние элементы) и кнопка перехода в полноэкранный раздел.

**Зачем:** Быстрый обзор «что происходит» и точка входа во все разделы. Карточки можно переставлять (drag-and-drop).

**Не дублирует** остальное: это только превью + навигация.

---

### 2.2 Полноэкранные виджеты (`/widget/:id`)

**Projects, Tasks, Files, Clients, Finances, Calendar, Messages, Analytics** — каждый открывается по своей ссылке и даёт **полный функционал** по своей области:

- списки, фильтры, поиск;
- создание/редактирование/удаление;
- канбан, календарь, таблицы — в зависимости от виджета.

**Смысл:** Здесь делается **глубокая работа** с проектами, задачами, файлами, клиентами и т.д.

---

### 2.3 Workspace (`/widget/workspace`)

**По задумке** Workspace — это **командный центр**: одна страница, где видно «всё по команде» без переключения между разделами.

- **Overview** — агрегированная сводка: активные проекты, просроченные задачи/деливерабли, ожидание фидбэка, предстоящие дедлайны, загрузки, нагрузка по людям, хранилище, активность клиентов. Это то, чего **нет** в виде одного экрана в других виджетах.
- **Остальные табы** (Team, Clients, Projects, Tasks, Deliverables, Files, Templates, Settings) — **краткие списки и быстрые действия** на одной странице, плюс ссылки/переходы в полные разделы (например, в Settings).

То есть:
- **Workspace** = «сводка + быстрый доступ» в одном месте.
- **Остальные виджеты** = полная работа с данными в своей зоне.

---

## 3. Почему кажется, что «всё повторяется»

Да, одни и те же **данные** (проекты, клиенты, задачи, файлы) появляются и в Workspace, и в своих виджетах. Разница не в наборе данных, а в **роли экрана**:

| Где | Роль | Уровень детализации |
|-----|------|----------------------|
| **Dashboard** | Превью + вход в разделы | Только счётчики/последнее |
| **Workspace** | Сводка по команде + быстрые действия | Списки, KPI, переходы в полные разделы |
| **Projects / Tasks / Files / …** | Полная работа с областью | Полные списки, CRUD, фильтры, спец. UI |

Если Workspace использовать только как «ещё один список проектов/задач», он действительно выглядит дублем. Его смысл проявляется, когда:

1. **Сначала смотришь Overview** — один экран с просрочками, фидбэком, нагрузкой, хранилищем.
2. **При необходимости** переходишь по табу (Team, Projects, …) для краткого списка или быстрого действия (например, «Сохранить проект как шаблон»).
3. **Для глубокой работы** уходишь в соответствующий виджет (Projects, Tasks и т.д.) по ссылке или через сайдбар.

---

## 4. Что можно улучшить, чтобы суть была яснее

1. **Сделать Overview главным по умолчанию** в Workspace и визуально выделить его как «сводку по команде», а остальные табы — как «быстрый доступ».
2. **Добавить явные призывы к действию** в табах Workspace: «Открыть все проекты →», «Перейти в Tasks →», чтобы было очевидно: здесь — обзор, там — полный раздел.
3. **В сайдбаре или на главной** коротко пояснять разницу: например, подпись к Workspace — «Сводка по команде», к Projects — «Проекты и доски».
4. **Сократить или переименовать табы** в Workspace, если какие-то из них по сути дублируют виджеты без добавления ценности (например, оставить только Overview + Team + Settings + быстрые ссылки на остальное).

---

## 5. Кратко: суть платформы

- **Продукт:** хаб для команд с проектами и клиентами: внутренняя работа (проекты, задачи, файлы, финансы) + портал для клиентов и обратная связь по деливераблицам.
- **Dashboard:** обзор и навигация по разделам (превью-карточки).
- **Workspace:** командный центр — сводка (Overview) и быстрый доступ к тем же данным на одной странице.
- **Остальные виджеты:** полный функционал по своей области.

Повтор данных между Workspace и виджетами задуман как **разный уровень детализации и разные сценарии** (сводка vs глубокая работа), а не как два независимых набора фич. Уточнение UX и подписей поможет пользователям сразу видеть эту разницу.
